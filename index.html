Cancelable Promises

Introduction#

Background explanatory material for this specification can be found in the domenic/cancelable-promise repository. See also the issues, which contain many ongoing discussions and debates.

1Promise Objects#

A Promise is an object that is used as a placeholder for the eventual results of a deferred (and possibly asynchronous) computation.

Any Promise object is in one of three mutually exclusive states: fulfilled, rejected, and pending:

  • A promise p is fulfilled if p.then(f, r) will immediately enqueue a Job to call the function f.
  • A promise p is rejected if p.then(f, r) will immediately enqueue a Job to call the function r.
  • A promise is pending if it is neither fulfilled nor rejected.

A promise is said to be settled if it is not pending, i.e. if it is either fulfilled or rejected.

A promise is resolved if it is settled or if it has been “locked in” to match the state of another promise. Attempting to resolve or reject a resolved promise has no effect. A promise is unresolved if it is not resolved. An unresolved promise is always in the pending state. A resolved promise may be pending, fulfilled or rejected.

A rejected promise is canceled if its rejection reason has a [[CancelBrand]] internal slot, and is errored otherwise.

1.1Promise Abstract Operations#

1.1.1PromiseReaction Records#

The PromiseReaction is a Record value used to store information about how a promise should react when it becomes resolved or rejected with a given value. PromiseReaction records are created by the PerformPromiseThen abstract operation, and are used by a PromiseReactionJob.

PromiseReaction records have the fields listed in Table 1.

1.1.2Promise Resolve Functions#

A promise resolve function is an anonymous built-in function that has [[Promise]] and [[AlreadyResolved]] internal slots.

When a promise resolve function F is called with argument resolution, the following steps are taken:

  1. Assert: F has a [[Promise]] internal slot whose value is an Object.
  2. Let promise be F.[[Promise]].
  3. Let alreadyResolved be F.[[AlreadyResolved]].
  4. If alreadyResolved.[[Value]] is true, return undefined.
  5. Set alreadyResolved.[[Value]] to true.
  6. Return ResolvePromise(promise, resolution).
  7. (all previous steps have been moved to that abstract operation)

The length property of a promise resolve function is 1.

1.1.3ResolvePromise ( promise, resolution )#

When the ResolvePromise abstract operation is called with arguments promise and resolution, the following steps are taken:

  1. If SameValue(resolution, promise) is true, then
    1. Let selfResolutionError be a newly created TypeError object.
    2. Return RejectPromise(promise, selfResolutionError).
  2. If Type(resolution) is not Object, then
    1. Return FulfillPromise(promise, resolution).
  3. Let then be Get(resolution, "then").
  4. If then is an abrupt completion, then
    1. Return RejectPromise(promise, then.[[Value]]).
  5. Let thenAction be then.[[Value]].
  6. If IsCallable(thenAction) is false, then
    1. Return FulfillPromise(promise, resolution).
  7. Perform EnqueueJob("PromiseJobs", PromiseResolveThenableJob, « promise, resolution, thenAction »).

1.1.4RejectPromise ( promise, reason )#

When the RejectPromise abstract operation is called with arguments promise and reason, the following steps are taken:

  1. Assert: The value of promise.[[PromiseState]] is "pending".
  2. Let reactions be promise.[[PromiseRejectReactions]].
  3. Set promise.[[PromiseResult]] to reason.
  4. Set promise.[[PromiseFulfillReactions]] to undefined.
  5. Set promise.[[PromiseRejectReactions]] to undefined.
  6. Set promise.[[PromiseState]] to "rejected".
  7. If promise.[[PromiseIsHandled]] is false and reason has no [[CancelBrand]] internal slot, perform HostPromiseRejectionTracker(promise, "reject").
  8. Return TriggerPromiseReactions(reactions, reason).

1.1.5HostPromiseRejectionTracker ( promise, operation )#

HostPromiseRejectionTracker is an implementation-defined abstract operation that allows host environments to track promise rejections.

An implementation of HostPromiseRejectionTracker must complete normally in all cases. The default implementation of HostPromiseRejectionTracker is to unconditionally return an empty normal completion.

Note 1

HostPromiseRejectionTracker is called in two scenarios:

  • When a promise is rejected with an object that is not a Cancel instance without any handlers, it is called with its operation argument set to "reject".
  • When a handler is added to such a rejected promise for the first time, it is called with its operation argument set to "handle".

A typical implementation of HostPromiseRejectionTracker might try to notify developers of unhandled rejections, while also being careful to notify them if such previous notifications are later invalidated by new handlers being attached.

 Note 2

If operation is "handle", an implementation should not hold a reference to promise in a way that would interfere with garbage collection. An implementation may hold a reference to promise if operation is "reject", since it is expected that rejections will be rare and not on hot code paths.

1.2Promise Jobs#

1.2.1PromiseReactionJob ( reaction, argument )#

The job PromiseReactionJob with parameters reaction and argument applies the appropriate handler to the incoming value, and uses the handler's return value to resolve or reject the derived promise associated with that handler.

  1. Assert: reaction is a PromiseReaction Record.
  2. Let promiseCapability be reaction.[[Capability]].
  3. Let type be reaction.[[Type]].
  4. Let handler be reaction.[[Handler]].
  5. If type is "Else" and argument has a [[CancelBrand]] internal slot, set handler to undefined.
  6. If handler is undefined, then
    1. If type is "Fulfill", let handlerResult be NormalCompletion(argument).
    2. Otherwise, type is "Reject" or "Else". Let handlerResult be Completion {[[Type]]: throw, [[Value]]: argument, [[Target]]: empty}.
  7. Else, let handlerResult be Call(handler, undefined, « argument »).
  8. If handlerResult is an abrupt completion, then
    1. Let status be Call(promiseCapability.[[Reject]], undefined, « handlerResult.[[Value]] »).
  9. Else,
    1. Let status be Call(promiseCapability.[[Resolve]], undefined, « handlerResult.[[Value]] »).
  10. Return Completion(status).

1.3Properties of the Promise Constructor#

1.3.1Promise.withCancelToken ( cancelToken, executor )#

Note

A different name for this operation, or even the possibility of merging it into the Promise constructor, is under discussion in #38.

The withCancelToken function returns a new promise created in a similar manner to the Promise constructor, but with additional abilities related to promise cancelation. If the given cancelToken becomes canceled, the returned promise will become canceled in the same way, and the function returned by executor (known as the "cancel action") will be called.

  1. Let C be the this value.
  2. If cancelToken is not undefined and ! IsCancelToken(cancelToken) is false, throw a TypeError exception.
  3. If IsCallable(executor) is false, throw a TypeError exception.
  4. Let promiseCapability be ? NewPromiseCapability(C).
  5. Let completion be Call(executor, undefined, « promiseCapability.[[Resolve]], promiseCapability.[[Reject]] »).
  6. If completion is an abrupt completion, then
    1. Perform ? Call(promiseCapability.[[Reject]], undefined, « completion.[[Value]] »).
    2. Return promise.
  7. Let cancelAction be completion.[[Value]].
  8. If cancelAction is undefined, return promise.
  9. If IsCallable(cancelAction) is false,
    1. Let badCancelAction be a new TypeError exception.
    2. Perform ? Call(promiseCapability.[[Reject]], undefined, « badCancelAction »).
    3. Return promise.
  10. If cancelToken is undefined, return promise.
  11. Let throwawayCapability be ! NewPromiseCapability(%Promise%).
  12. Let onFulfilled be a new built-in function object as defined in `Promise.withCancelToken` Cancelation Reaction Functions.
  13. Set onFulfilled.[[CancelAction]] to cancelAction.
  14. Set onFulfilled.[[Reject]] to promiseCapability.[[Reject]].
  15. Perform ! PerformPromiseThen(cancelToken.[[CancelTokenPromise]], onFulfilled, undefined, throwawayCapability).
  16. Return promise.

1.3.1.1Promise.withCancelToken Cancelation Reaction Functions#

A Promise.withCancelToken cancelation reaction function is an anonymous built-in function that is used to react to a cancel token's cancelation in the Promise.withCancelToken function. Each Promise.withCancelToken cancelation reaction function has [[CancelAction]] and [[Reject]] internal slots.

When a Promise.withCancelToken cancelation reaction function F is called with argument cancelation, the following steps are taken:

  1. Let cancelAction be F.[[CancelAction]].
  2. Let reject be F.[[Reject]].
  3. Let result be Call(cancelAction, undefined, « cancelation »).
  4. If result is an abrupt completion, perform ? Call(reject, undefined, « result.[[Value]] »).
  5. Otherwise, perform ? Call(reject, undefined, « cancelation »).
  6. Return undefined.

1.4Properties of the Promise Prototype Object#

The Promise prototype object is the intrinsic object %PromisePrototype%. The value of the [[Prototype]] internal slot of the Promise prototype object is the intrinsic object %ObjectPrototype%. The Promise prototype object is an ordinary object. It does not have a [[PromiseState]] internal slot or any of the other internal slots of Promise instances.

1.4.1Promise.prototype.else ( onErrored )#

When the else method is called with argument onErrored, the following steps are taken:

  1. Let promise be the this value.
  2. If IsPromise(promise) is false, throw a TypeError exception.
  3. Let C be ? SpeciesConstructor(promise, %Promise%).
  4. Let resultCapability be ? NewPromiseCapability(C).
  5. Return PerformPromiseThen(promise, undefined, onErrored, resultCapability, "Else").

1.5Promise.prototype.then ( onFulfilled, onRejected )#

When the then method is called with arguments onFulfilled and onRejected, the following steps are taken:

  1. Let promise be the this value.
  2. If IsPromise(promise) is false, throw a TypeError exception.
  3. Let C be ? SpeciesConstructor(promise, %Promise%).
  4. Let resultCapability be ? NewPromiseCapability(C).
  5. Return PerformPromiseThen(promise, onFulfilled, onRejected, resultCapability, "Rejected").

1.5.1PerformPromiseThen ( promise, onFulfilled, onRejected, resultCapability, rejectReactionType )#

The abstract operation PerformPromiseThen performs the “then” operation on promise using onFulfilled and onRejected as its settlement actions (with onRejected's behavior being further modified by rejectReactionType). The result is resultCapability's promise.

  1. Assert: IsPromise(promise) is true.
  2. Assert: resultCapability is a PromiseCapability Record.
  3. If IsCallable(onFulfilled) is false, then
    1. Let onFulfilled be undefined.
  4. If IsCallable(onRejected) is false, then
    1. Let onRejected be undefined.
  5. Let fulfillReaction be the PromiseReaction { [[Capability]]: resultCapability, [[Type]]: "Fulfill", [[Handler]]: onFulfilled }.
  6. Let rejectReaction be the PromiseReaction { [[Capability]]: resultCapability, [[Type]]: "Reject"rejectReactionType, [[Handler]]: onRejected }.
  7. If promise.[[PromiseState]] is "pending", then
    1. Append fulfillReaction as the last element of the List that is promise.[[PromiseFulfillReactions]].
    2. Append rejectReaction as the last element of the List that is promise.[[PromiseRejectReactions]].
  8. Else if promise.[[PromiseState]] is "fulfilled", then
    1. Let value be promise.[[PromiseResult]].
    2. Perform EnqueueJob("PromiseJobs", PromiseReactionJob, « fulfillReaction, value »).
  9. Else,
    1. Assert: The value of promise.[[PromiseState]] is "rejected".
    2. Let reason be promise.[[PromiseResult]].
    3. If promise.[[PromiseIsHandled]] is false and reason does not have a [[CancelBrand]] internal slot, perform HostPromiseRejectionTracker(promise, "handle").
    4. Perform EnqueueJob("PromiseJobs", PromiseReactionJob, « rejectReaction, reason »).
  10. Set promise.[[PromiseIsHandled]] to true.
  11. Return resultCapability.[[Promise]].

2Cancel Objects#

Instances of Cancel objects are thrown as exceptions when an operations has been canceled. They do not indicate a runtime error, and are treated differently from other exceptions in several places in this specification.

2.1The Cancel Constructor#

The Cancel constructor is the %Cancel% intrinsic object and the initial value of the Cancel property of the global object. When Cancel is called as a function rather than as a constructor, it creates and initializes a new Cancel object. Thus the function call Cancel(…) is equivalent to the object creation expression new Cancel(…) with the same arguments.

The Cancel constructor is designed to be subclassable. It may be used as the value of an extends clause of a class definition. Subclass constructors that intend to inherit the specified Cancel behaviour must include a super call to the Cancel constructor to create and initialize subclass instances with a [[CancelBrand]] internal slot.

2.1.1Cancel ( message )#

When the Cancel function is called with argument message, the following steps are taken:

  1. If NewTarget is undefined, let newTarget be the active function object, else let newTarget be NewTarget.
  2. Let O be ? OrdinaryCreateFromConstructor(newTarget, "%CancelPrototype%", « [[CancelBrand]] »).
  3. If message is not undefined, then
    1. Let msg be ? ToString(message).
    2. Let msgDesc be the PropertyDescriptor { [[Value]]: msg, [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: true }.
    3. Perform ! DefinePropertyOrThrow(O, "message", msgDesc).
  4. Return O.

2.2Properties of the Cancel Constructor#

The value of the [[Prototype]] internal slot of the Cancel constructor is the intrinsic object %FunctionPrototype%.

The Cancel constructor has the following properties:

2.2.1Cancel.prototype#

The initial value of Cancel.prototype is the intrinsic object %CancelPrototype%.

This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

2.3Properties of the Cancel Prototype Object#

The Cancel prototype object is the intrinsic object %CancelPrototype%. The Cancel prototype object is an ordinary object. It is not a Cancel instance and does not have a [[CancelBrand]] internal slot.

The value of the [[Prototype]] internal slot of the Cancel prototype object is the intrinsic object %ObjectPrototype%.

2.3.1Cancel.prototype.constructor#

The initial value of Cancel.prototype.constructor is the intrinsic object %Cancel%.

2.3.2Cancel.prototype.toString ( )#

The following steps are taken:

  1. Let O be the this value.
  2. If Type(O) is not Object, throw a TypeError exception.
  3. Let msg be ? Get(O, "message").
  4. If msg is undefined, let msg be the empty String; otherwise let msg be ? ToString(msg).
  5. If msg is the empty String, return "Cancel".
  6. Return the result of concatenating "Cancel: " and msg.

2.4Properties of Cancel Instances#

Cancel instances are ordinary objects that inherit properties from the Cancel prototype object and have a [[CancelBrand]] internal slot whose value is undefined. The only use of [[CancelBrand]] is to identify Cancel instances; its actual value is not important and it does not store any data.

3CancelToken Objects#

A cancel token is an object that allows its consumer to monitor and respond to requests for cancelation of an ongoing operation.

3.1CancelToken Abstract Operations#

3.1.1CancelCancelToken ( cancelToken, message )#

When the CancelCancelToken abstract operation is called with arguments cancelToken and message, the following steps are taken:

  1. Let promise be cancelToken.[[CancelTokenPromise]].
  2. Let cancelation be ? Construct(%Cancel%, « cancelMessage »).
  3. Perform ! ResolvePromise(promise, cancelation).

3.1.2CancelTokenCancelation ( cancelToken )#

The abstract operation CancelTokenCancelation returns the cancel token's cancelation.

  1. Assert: ! IsCancelToken(cancelToken) is true.
  2. Assert: ! CancelTokenIsRequested(cancelToken) is true.
  3. Return cancelToken.[[CancelTokenPromise]].[[PromiseResult]].

3.1.3CancelTokenIsRequested ( cancelToken )#

The abstract operation CancelTokenIsRequested returns true if cancelation has been requested for the given cancel token, or false otherwise.

  1. Assert: ! IsCancelToken(cancelToken) is true.
  2. If cancelToken.[[CancelTokenPromise]].[[PromiseState]] is "fulfilled", return true.
  3. Return false.

3.1.4IsCancelToken ( x )#

The abstract operation IsCancelToken checks for the cancel token brand on an object.

  1. If Type(x) is not Object, return false.
  2. If x does not have a [[CancelTokenPromise]] internal slot, return false.
  3. Return true.

3.1.5NewCancelTokenSource ( C )#

The abstract operation NewCancelTokenSource takes a constructor function, and attempts to use that constructor function in the fashion of the built-in CancelToken constructor to create a cancel token and extract its cancel function. The cancel token plus its cancel function is used to initialize a new Record which is returned.

  1. If IsConstructor(C) is false, throw a TypeError exception.
  2. Let source be a new Record { [[Token]]: undefined, [[Cancel]]: undefined }.
  3. Let executor be a new built-in function object as defined in GetCancelExecutor Functions.
  4. Set executor.[[Source]] to source.
  5. Let token be ? Construct(C, « executor »).
  6. Set source.[[Token]] to token.
  7. Return source.

3.1.5.1GetCancelExecutor Functions#

A GetCancelExecutor function is an anonymous built-in function that has a [[Source]] internal slot.

When a GetCancelExecutor function F is called with argument cancel, the following steps are taken:

  1. Assert: F has a [[Source]] internal slot whose value is a Record containing a field [[Cancel]].
  2. Let source be F.[[Source]].
  3. Set source.[[Cancel]] to cancel.
  4. Return undefined.

The length property of a GetCancelExecutor function is 1.

3.2The CancelToken Constructor#

The CancelToken constructor is the %CancelToken% intrinsic object and the initial value of the CancelToken property of the global object. When called as a constructor, it creates and initializes as a new CancelToken object. CancelToken is not intended to be called as a function and will throw an exception when called in that manner.

The CancelToken constructor is designed to be subclassable. It may be used as the value in an extends clause of a class definition. Subclass constructors that intend to inherit the specified CancelToken behaviour must include a super call to the CancelToken constructor to create and initialize the subclass instance with the internal state necessary to support the CancelToken and CancelToken.prototype built-in methods.

3.2.1CancelToken ( executor )#

When the CancelToken function is called with argument executor, the following steps are taken:

  1. If NewTarget is undefined, throw a TypeError exception.
  2. If IsCallable(executor) is false, throw a TypeError exception.
  3. Let cancelToken be ? OrdinaryCreateFromConstructor(NewTarget, "%CancelTokenPrototype", « [[CancelTokenPromise]] »).
  4. Let dummyExecutor be an anonymous built-in function of no arguments that does nothing besides return undefined.
  5. Let promise be ! Construct(%Promise%, « executor »).
  6. Set cancelToken.[[CancelTokenPromise]] to promise.
  7. Let cancel be a new built-in function object as defined in Canceler Functions.
  8. Set cancel.[[CancelToken]] to cancelToken.
  9. Perform ? Call(executor, undefined, « cancel »).
  10. Return cancelToken.
Note

The executor argument must be a function object. It is called to vend the ability to send the cancelation token into a canceled state, via its single argument, cancel.

The cancel function that is passed to an executor function accepts a single argument. The executor code may eventually call the cancel function indicate it wishes to request cancelation. The argument passed to the cancel function represents the cancelation. If it is not a "thenable" value, cancelation will be requested immediately; otherwise, cancelation will be requested once the passed thenable fulfills.

3.2.1.1Canceler Functions#

A Canceler function is an anonymous built-in function that has a [[CancelToken]] internal slot.

When a Canceler function F is called with argument cancelMessage, the following steps are taken:

  1. Assert: F has a [[CancelToken]] internal slot whose value is an instance of the intrinsic %CancelToken%.
  2. Let cancelToken be F.[[CancelToken]].
  3. Return CancelCancelToken(cancelToken, cancelMessage).

3.3Properties of the CancelToken Constructor#

The value of the [[Prototype]] internal slot of the CancelToken constructor is the intrinsic object %CancelTokenPrototype%.

The CancelToken constructor has the following properties:

3.3.1CancelToken.source ( )#

The source() function returns an object with two properties, token and cancel, which respectively give a newly-created cancel token, and the ability to request cancelation for that token. It is a convenient alternative to the CancelToken constructor.

  1. Let C be the this value.
  2. Let source be ? NewCancelTokenSource(C).
  3. Let result be ! ObjectCreate(%ObjectPrototype%).
  4. Perform ! CreateDataProperty(result, "token", source.[[Token]]).
  5. Perform ! CreateDataProperty(result, "cancel", source.[[Cancel]]).
  6. Return result.

3.3.2CancelToken.race ( iterable )#

The race function returns a new cancel token which becomes canceled in the same way as the first passed cancel token to become canceled. If iterable contains any elements that are not cancel tokens, the function throws a TypeError exception.

  1. Let C be the this value.
  2. Let source be ? NewCancelTokenSource(C).
  3. Let tokens be ? IterableToList(iterable).
  4. For each element token of tokens,
    1. If ! IsCancelToken(token) is false, throw a TypeError exception.
  5. For each element token of tokens,
    1. Assert: ! IsCancelToken(token) is true.
    2. Let throwawayCapability be ! NewPromiseCapability(%Promise%).
    3. Perform ! PerformPromiseThen(token.[[CancelTokenPromise]], source.[[Cancel]], undefined, throwawayCapability).
  6. Return source.[[Token]].

3.3.3CancelToken.prototype#

The initial value of CancelToken.prototype is the intrinsic object %CancelTokenPrototype%.

This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

3.4Properties of the CancelToken Prototype Object#

The CancelToken prototype object is the intrinsic object %CancelTokenPrototype%. The value of the [[Prototype]] internal slot of the CancelToken prototype object is the intrinsic object %ObjectPrototype%. The CancelToken prototype object is an ordinary object. It does not have a [[CancelTokenPromise]] internal slot or any of the other internal slots of CancelToken instances.

3.4.1CancelToken.prototype.throwIfRequested ( )#

When the throwIfRequested method is called, the following steps are taken:

  1. Let T be the this value.
  2. If ! IsCancelToken(T) is false, throw a TypeError exception.
  3. If ! CancelTokenIsRequested(T) is true, return Completion { [[Type]]: throw, [[Value]]: CancelTokenCancelation(T), [[Target]]: empty }.
  4. Return undefined.

3.4.2CancelToken.prototype.constructor#

The initial value of CancelToken.prototype.constructor is the intrinsic object %CancelToken%.

3.4.3get CancelToken.prototype.promise#

CancelToken.prototype.promise is an accessor property whose set accessor is undefined. Its get accessor function performs the following steps:

  1. Let T be the this value.
  2. If ! IsCancelToken(T) is false, throw a TypeError exception.
  3. Return T.[[CancelTokenPromise]].

3.4.4get CancelToken.prototype.requested#

CancelToken.prototype.requested is an accessor property whose set accessor is undefined. Its get accessor function performs the following steps:

  1. Let T be the this value.
  2. If ! IsCancelToken(T) is false, throw a TypeError exception.
  3. Return ! CancelTokenIsRequested(T).

3.4.5CancelToken.prototype [ @@toStringTag ]#

The initial value of the @@toStringTag property is the String value "CancelToken".

This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }.

3.5Properties of CancelToken Instances#

CancelToken instances are ordinary objects that inherit properties from the CancelToken prototype object (the intrinsic, %CancelTokenPrototype%). CancelToken instances are initially created with the internal slots described in Table 2.

The try statement encloses a block of code in which an exceptional condition can occur, such as a runtime error or a throw statement. The catch clause provides the exception-handling code. When a catch clause catches an exception, its CatchParameter is bound to that exception. The else clause provides error-handling code. It behaves identically to a catch clause, except that it does not catch cancelations (i.e. Cancel objects).

The abstract operation MakeAwaitCancelTokenReference with argument strict performs the following steps:

  2. If runningContext does not have a CancelToken component, throw a ReferenceError exception.

5.3The Reference Specification Type#

Note

The Reference type is used to explain the behaviour of such operators as delete, typeof, the assignment operators, the super keyword, the await.cancelToken meta-property, and other language features. For example, the left-hand operand of an assignment is expected to produce a reference.

A Reference is a resolved name or property binding. A Reference consists of three components, the base value component, the referenced name component, and the Boolean-valued strict reference flag. The base value component is either undefined, an Object, a Boolean, a String, a Symbol, a Number, an Environment Record, or an execution context. A base value component of undefined indicates that the Reference could not be resolved to a binding. The referenced name component is a String or Symbol value.

A Super Reference is a Reference that is used to represents a name binding that was expressed using the super keyword. A Super Reference has an additional thisValue component, and its base value component will never be an Environment Record.

An Await Cancel Token Reference is a Reference that is used to represent the assignable binding expressed using the await.cancelToken meta-property. An Await Cancel Token Reference's base value component is always an execution context.

The following abstract operations are used in this specification to access the components of references:

  • GetBase(V). Returns the base value component of the reference V.
  • GetReferencedName(V). Returns the referenced name component of the reference V.
  • IsStrictReference(V). Returns the strict reference flag of the reference V.
  • HasPrimitiveBase(V). Returns true if Type(V's base value component) is Boolean, String, Symbol, or Number.
  • IsPropertyReference(V). Returns true if either the base value component of the reference V is an object or HasPrimitiveBase(V) is true; otherwise returns false.
  • IsUnresolvableReference(V). Returns true if the base value component of the reference V is undefined; otherwise returns false.
  • IsSuperReference(V). Returns true if the reference V has a thisValue component.
  • IsAwaitCancelTokenReference(V). Returns true if the base value component of the reference V is an execution context.

The following abstract operations are used in this specification to operate on references:

5.3.1GetValue (V)#

  1. ReturnIfAbrupt(V).
  2. If Type(V) is not Reference, return V.
  3. Let base be GetBase(V).
  4. If IsUnresolvableReference(V) is true, throw a ReferenceError exception.
  5. If IsPropertyReference(V) is true, then
    1. If HasPrimitiveBase(V) is true, then
      1. Assert: In this case, base will never be null or undefined.
      2. Let base be ! ToObject(base).
    2. Return ? base.[[Get]](GetReferencedName(V), GetThisValue(V)).
  6. If IsAwaitCancelTokenReference(V) is true, then
    1. Return the value of base's CancelToken component.
  7. Else base must be an Environment Record,
    1. Return ? base.GetBindingValue(GetReferencedName(V), IsStrictReference(V)) (see 8.1.1).
Note

The object that may be created in step 5.a.ii is not accessible outside of the above abstract operation and the ordinary object [[Get]] internal method. An implementation might choose to avoid the actual creation of the object.

5.3.2PutValue (V, W)#

  1. ReturnIfAbrupt(V).
  2. ReturnIfAbrupt(W).
  3. If Type(V) is not Reference, throw a ReferenceError exception.
  4. Let base be GetBase(V).
  5. If IsUnresolvableReference(V) is true, then
    1. If IsStrictReference(V) is true, then
      1. Throw a ReferenceError exception.
    2. Let globalObj be GetGlobalObject().
    3. Return ? Set(globalObj, GetReferencedName(V), W, false).
  6. Else if IsPropertyReference(V) is true, then
    1. If HasPrimitiveBase(V) is true, then
      1. Assert: In this case, base will never be null or undefined.
      2. Set base to ! ToObject(base).
    2. Let succeeded be ? base.[[Set]](GetReferencedName(V), W, GetThisValue(V)).
    3. If succeeded is false and IsStrictReference(V) is true, throw a TypeError exception.
    4. Return.
  7. Else if IsAwaitCancelTokenReference(V) is true, then
    1. If W is undefined,
      1. Set the value of base's CancelToken component to undefined.
    2. Otherwise,
      1. Set the value of base's CancelToken component to undefined.
      2. If ! IsCancelToken(W) is false, throw a TypeError exception.
      3. Set base's CancelToken component to W.
  8. Else base must be an Environment Record,
    1. Return ? base.SetMutableBinding(GetReferencedName(V), W, IsStrictReference(V)) (see 8.1.1).
Note

The object that may be created in step 6.a.ii is not accessible outside of the above algorithm and the ordinary object [[Set]] internal method. An implementation might choose to avoid the actual creation of that object.

5.4Async Functions#

5.4.1AsyncFunctionStart ( promiseCapability, asyncFunctionBody )#

  1. Let runningContext be the running execution context.
  2. Let asyncContext be a copy of runningContext.
  3. Set the CancelToken component of asyncContext to undefined.
  4. Set the code evaluation state of asyncContext such that when evaluation is resumed for that execution context the following steps will be performed:
    1. Let result be the result of evaluating asyncFunctionBody.
    2. Assert: If we return here, the async function either threw an exception or performed an implicit or explicit return; all awaiting is done.
    3. Remove asyncContext from the execution context stack and restore the execution context that is at the top of the execution context stack as the running execution context.
    4. If result.[[type]] is normal, then
      1. Perform ! Call(promiseCapability.[[Resolve]], undefined, « undefined »).
    5. Else if result.[[type]] is return, then
      1. Perform ! Call(promiseCapability.[[Resolve]], undefined, « result.[[Value]] »).
    6. Else,
      1. Assert: result.[[type]] is throw.
      2. Perform ! Call(promiseCapability.[[Reject]], undefined, « result.[[Value]] »).
    7. Return.
  5. Push asyncContext onto the execution context stack; asyncContext is now the running execution context.
  6. Resume the suspended evaluation of asyncContext. Let result be the value returned by the resumed computation.
  7. Assert: When we return here, asyncContext has already been removed from the execution context stack and runningContext is the currently running execution context.
  8. Assert: result is a normal completion with a value of undefined. The possible sources of completion values are AsyncFunctionAwait or, if the async function doesn't await anything, the step 3.7 above.
  9. Return.

5.4.2AsyncFunctionAwait ( value )#

  1. Let asyncContext be the running execution context.
  2. Let cancelToken be asyncContext's CancelToken component.
  3. If cancelToken is not undefined, and ! CancelTokenIsRequested(cancelToken) is true, return Completion { [[Type]]: throw, [[Value]]: CancelTokenCancelation(cancelToken), [[Target]]: empty }.
  4. Let promiseCapability be ! NewPromiseCapability(%Promise%).
  5. Let resolveResult bePerform ! Call(promiseCapability.[[Resolve]], undefined, « value »).
  6. Let onFulfilled be a new built-in function object as defined in AsyncFunction Awaited Fulfilled Functions.
  7. Let onRejected be a new built-in function object as defined in AsyncFunction Awaited Rejected Functions.
  8. Set onFulfilled and onRejected's [[AsyncContext]] internal slots to asyncContext.
  9. Let alreadyResumed be a new Record { [[Value]]: false }.
  10. Set onFulfilled and onReject's [[AlreadyResumed]] internal slots to alreadyResumed.
  11. If cancelToken is not undefined,
    1. Let cancelTokenThrowawayCapability be ! NewPromiseCapability(%Promise%).
    2. Perform ! PerformPromiseThen(cancelToken.[[CancelTokenPromise]], onRejected, undefined, cancelTokenThrowawayCapability).
  12. Let throwawayCapability be NewPromiseCapability(%Promise%).
  13. Perform ! PerformPromiseThen(promiseCapability.[[Promise]], onFulfilled, onRejected, throwawayCapability).
  14. Remove asyncContext from the execution context stack and restore the execution context that is at the top of the execution context stack as the running execution context.
  15. Set the code evaluation state of asyncContext such that when evaluation is resumed with a Completion resumptionValue the following steps will be performed:
    1. Return resumptionValue.
  16. Return.

5.4.2.1AsyncFunction Awaited Fulfilled Functions#

Function F is called with the parameter value.

An AsyncFunction awaited fulfilled function is an anonymous built-in function that has [[AsyncContext]] and [[AlreadyResumed]] internal slots.

When an AsyncFunction awaited fulfilled function F is called with argument value, the following steps are taken:

  1. Let alreadyResumed be F.[[AlreadyResumed]].
  2. If alreadyResumed.[[Value]] is true, return undefined.
  3. Set alreadyResumed.[[Value]] to true.
  4. Let asyncContext be the value of F's [[AsyncContext]] internal slotF.[[AsyncContext]].
  5. Let prevContext be the running execution context.
  6. Suspend prevContext.
  7. Push asyncContext onto the execution context stack; asyncContext is now the running execution context.
  8. Resume the suspended evaluation of asyncContext using NormalCompletion(value) as the result of the operation that suspended it. Let result be the value returned by the resumed computation.
  9. Assert: When we reach this step, asyncContext has already been removed from the execution context stack and prevContext is the currently running execution context.
  10. Return Completion(result).

5.4.2.2AsyncFunction Awaited Rejected Functions#

Function F is called with the parameter reason.

An AsyncFunction awaited rejected function is an anonymous built-in function that has [[AsyncContext]] and [[AlreadyResumed]] internal slots.

When an AsyncFunction awaited rejected function F is called with argument reason, the following steps are taken:

  1. Let alreadyResumed be F.[[AlreadyResumed]].
  2. If alreadyResumed.[[Value]] is true, return undefined.
  3. Set alreadyResumed.[[Value]] to true.
  4. Let asyncContext be the value of F's [[AsyncContext]] internal slotF.[[AsyncContext]].
  5. Let prevContext be the running execution context.
  6. Suspend prevContext.
  7. Push asyncContext onto the execution context stack; asyncContext is now the running execution context.
  8. Resume the suspended evaluation of asyncContext using Completion{[[type]]: throw, [[value]]: reason, [[target]]: empty} as the result of the operation that suspended it. Let result be the value returned by the resumed computation.
  9. Assert: When we reach this step, asyncContext has already been removed from the execution context stack and prevContext is the currently running execution context.
  10. Return Completion(result).
